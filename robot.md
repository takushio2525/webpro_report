# ロボ部品リスト管理システム

## 概要
ロボット製作に必要な部品の在庫，保管場所，コストを管理するシステム．

---

## データ構造 (Data Structure)
| フィールド | 説明 | 例 |
|-----------|------|-----|
| `id` | ID | 1, 2, 3 |
| `part_name` | 部品名 | サーボモータ, Arduino |
| `part_number` | 型番 | SG90, Uno R3 |
| `order_number` | 注文番号 | ORD-001 |
| `quantity` | 個数 | 5 |
| `price` | 単価 | 500円 |
| `sale_site` | 販売サイトリンク | https://example.com |
| `datasheet` | データシートリンク | https://example.com/datasheet.pdf |
| `memo` | メモ | 予備として購入 |

---

## 機能一覧

### 共通機能
- **データ管理**: 配列を用いた簡易データベース（サーバー再起動でリセットされる仕様で可）
- **一覧表示 (List)**: 登録されている全データを表形式などで表示する
- **詳細表示 (Detail)**: データの詳細情報を表示する
- **新規登録 (Create)**: 新しいデータを追加するフォームと処理
- **編集 (Edit)**: 既存のデータを修正するフォームと処理
- **削除 (Delete)**: 既存のデータを削除する処理

### 独自機能
- **リンクをクリックして飛べる**: リンクをクリックしたら購入ページに移動できる
- **発注リストの出力**: 発注リストを出力する
- **メカ部品，電子部品等のタブ分け表示**: パーツの種類ごとにタブで表示する
